
 Доступы и внешние зависимости (нужны для полного запуска)

В репозитории лежат экспортированные n8n-воркфлоу проекта AK-ES.

По причинам безопасности и затрат (ключи/лимиты) в репозиторий не включены доступы к приватной инфраструктуре.

Что нужно, чтобы запустить воркфлоу 
- База данных (Postgres/Neon): строка подключения и доступ к используемым схемам/таблицам, например:
  - `cdm_source_features`
  - `difference` (DIFF-таблицы)
  - таблицы/коллекции векторного хранилища (embeddings/documents)
  - любые другие таблицы, которые упоминаются в нодах воркфлоу
- Ключ LLM-провайдера (например, OpenAI API key), если AI-Agent ноды делают запросы к внешней модели.
- Доступ к vector store, если retrieval реализован через отдельный сервис/таблицу/БД.

 Примечание
Даже без ключей можно оценить систему по артефактам:
- входные примеры: `samples/`
- реальные выходы модулей (включая retrieval top-k): `demo_outputs/`
- архитектура: `workflows/`
- скриншоты/демо: техдокументация и/или папка `docs/`
